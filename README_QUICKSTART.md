# Быстрый старт

## Автоматический запуск (рекомендуется)

Запустите оба сервера одной командой:

```bash
start-all.bat
```

Этот скрипт:
- ✅ Запустит бэкенд на порту 9999
- ✅ Запустит фронтенд на порту 3000
- ✅ Откроет оба сервера в отдельных окнах
- ✅ Покажет статус запуска

## Ручной запуск

### Вариант 1: Отдельные скрипты

1. **Бэкенд** (в первом терминале):
   ```bash
   start-backend.bat
   ```

2. **Фронтенд** (во втором терминале):
   ```bash
   start-frontend.bat
   ```

### Вариант 2: Через npm (только фронтенд + бэкенд)

```bash
cd frontend
npm run dev:with-backend
```

Это запустит оба сервера в одном терминале.

## Проверка работы

1. Откройте браузер: http://localhost:3000
2. Статус должен показать "Остановлено" (не "Бэкенд недоступен")
3. Нажмите "Запустить нормализацию" для тестирования

## Остановка

- **Автоматический запуск**: Закройте окна серверов
- **Ручной запуск**: Нажмите `Ctrl+C` в каждом терминале

## Устранение проблем

### Бэкенд не запускается

- Проверьте Go: `go version`
- Проверьте порт: `netstat -ano | findstr :9999`
- Убедитесь, что файл `1c_data.db` существует

### Фронтенд не запускается

- Проверьте Node.js: `node --version`
- Установите зависимости: `cd frontend && npm install`
- Проверьте порт: `netstat -ano | findstr :3000`

### "Бэкенд недоступен" в интерфейсе

- Убедитесь, что бэкенд запущен
- Проверьте: `curl http://localhost:9999/health`
- Перезагрузите страницу в браузере
