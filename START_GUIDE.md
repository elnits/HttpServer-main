# Руководство по запуску системы

## Быстрый старт

### Вариант 1: Использование batch скриптов (Windows)

1. **Запустите бэкенд:**
   ```bash
   start-backend.bat
   ```
   Бэкенд будет доступен на `http://localhost:9999`

2. **Запустите фронтенд** (в новом окне терминала):
   ```bash
   start-frontend.bat
   ```
   Фронтенд будет доступен на `http://localhost:3000`

### Вариант 2: Ручной запуск

1. **Запуск бэкенда:**
   ```bash
   cd E:\HttpServer
   go run main_no_gui.go
   ```

2. **Запуск фронтенда** (в новом терминале):
   ```bash
   cd E:\HttpServer\frontend
   npm run dev
   ```

## Проверка работы

1. Откройте браузер и перейдите на `http://localhost:3000`
2. Если бэкенд запущен, вы увидите статус "Остановлено"
3. Если бэкенд не запущен, вы увидите статус "Бэкенд недоступен" с инструкцией по запуску

## Использование

1. На главной странице нажмите "Управление нормализацией"
2. На странице нормализации нажмите "Запустить нормализацию"
3. Наблюдайте за прогрессом в реальном времени:
   - Прогресс-бар показывает общий прогресс
   - Логи отображают детальную информацию
   - Статистика обновляется автоматически

## Устранение проблем

### Бэкенд не запускается

- Проверьте, что Go установлен: `go version`
- Проверьте, что порт 9999 свободен: `netstat -ano | findstr :9999`
- Проверьте наличие файла `1c_data.db` или `data.db`

### Фронтенд не запускается

- Проверьте, что Node.js установлен: `node --version`
- Установите зависимости: `cd frontend && npm install`
- Проверьте, что порт 3000 свободен

### Ошибка "Бэкенд недоступен"

- Убедитесь, что бэкенд запущен на порту 9999
- Проверьте подключение: `curl http://localhost:9999/health`
- Проверьте переменные окружения в `frontend/.env.local`

## Порты

- **3000** - Next.js фронтенд
