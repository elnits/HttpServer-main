# Перезапуск Backend сервера

## Проблема
Backend сервер возвращает 404 для новых endpoints (`/api/clients`), потому что он не был перезапущен после добавления новых маршрутов.

## Решение

### Вариант 1: Использовать скрипт (Windows)
```bash
start-backend.bat
```

### Вариант 2: Вручную
1. Остановите текущий backend сервер (Ctrl+C в терминале, где он запущен)
2. Запустите заново:
   ```bash
   go run main_no_gui.go
   ```
   
   Или если используете скомпилированный файл:
   ```bash
   go build -o httpserver.exe .
   ./httpserver.exe
   ```

### Вариант 3: Использовать скрипт для запуска всего
```bash
start-all.bat
```
Этот скрипт запустит и backend, и frontend.

## Проверка
После перезапуска проверьте, что endpoints работают:
```bash
curl http://localhost:9999/api/clients
```

Должен вернуться список клиентов (пустой массив `[]` если клиентов нет) вместо 404.

## Важно
После любых изменений в `server/server.go` или добавления новых маршрутов необходимо перезапустить backend сервер!
